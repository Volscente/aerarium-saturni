## Architecture

```txt
Developer: alembic upgrade head
      │  (run from backend/)
      ▼
backend/alembic/env.py
  ├── imports Base.metadata ◄── src/backend/models.py
  │     (Transaction, Etf, EtfHolding, EtfPriceHistory)
  └── sync engine: create_engine(DATABASE_URL, poolclass=NullPool)
      │  psycopg3 sync mode — same URL prefix, no asyncio.run() wrapper
      ▼
backend/alembic/versions/001_create_etf_tables.py
  ├── upgrade()   → CREATE TABLE etfs → etf_holdings → etf_price_history
  └── downgrade() → DROP TABLE etf_price_history → etf_holdings → etfs
      │
      ▼
PostgreSQL schema result:
  etfs  ──1:N──► etf_holdings      (etf_id FK, ON DELETE CASCADE)
        └─1:N──► etf_price_history  (etf_id FK, ON DELETE CASCADE,
                                     composite B-Tree index on (etf_id, timestamp DESC))
```

### Why psycopg3 sync mode for Alembic

Alembic's migration runner is synchronous. The async `create_async_engine` in `db.py` cannot be used by Alembic without an `asyncio.run()` wrapper. psycopg3 (`psycopg[binary]`) natively supports both async and sync modes under the same `postgresql+psycopg://` URL prefix — `env.py` uses `create_engine` (not `create_async_engine`) with the same `DATABASE_URL`. No separate driver or alternate connection string is needed.

### Key Functions

```python
def upgrade() -> None:
    """Create etfs, etf_holdings, and etf_price_history tables with all constraints and indexes.

    Creates tables in dependency order (parent before children) so foreign key references
    resolve at DDL time. Indexes are added immediately after each table so the migration
    is fully self-contained and reversible.

    Tables created:
        etfs: Parent record with UUID PK, UNIQUE ticker/isin, four JSONB distribution
              columns, and scalar ETF metadata. GIN indexes on geographical_distribution
              and sector_distribution for containment queries.
        etf_holdings: Constituent positions linked to etfs via etf_id FK (ON DELETE CASCADE).
        etf_price_history: Price snapshots linked to etfs via etf_id FK (ON DELETE CASCADE)
                           with a composite B-Tree index on (etf_id, timestamp DESC) for
                           O(1) latest-price lookups.
    """
```

```python
def downgrade() -> None:
    """Drop etf_price_history, etf_holdings, and etfs in reverse dependency order.

    Drops child tables before the parent to avoid FK constraint violations at DDL time.
    Indexes on each table are dropped implicitly when their table is dropped.
    The transactions table (managed by create_all, not Alembic) is not touched.
    """
```

```python
def run_migrations_online() -> None:
    """Run migrations in online mode using a synchronous psycopg3 connection.

    Reads DATABASE_URL from the environment (injected via config.set_main_option in
    the env.py module body before this function is called). Uses NullPool so no
    connections are held open after the migration context exits.

    Raises:
        KeyError: If DATABASE_URL is not set in the environment before env.py is imported.
        sqlalchemy.exc.OperationalError: If PostgreSQL is unreachable at migration time.
    """
```

### Data Models / Schemas

```python
class Etf(Base):
    __tablename__ = "etfs"

    id: Mapped[uuid.UUID] = mapped_column(primary_key=True, default=uuid.uuid4)
    ticker: Mapped[str] = mapped_column(String(20), unique=True, nullable=False)
    isin: Mapped[str] = mapped_column(String(12), unique=True, nullable=False)
    name: Mapped[str] = mapped_column(String(200), nullable=False)
    issuer: Mapped[str] = mapped_column(String(100), nullable=False)
    asset_class: Mapped[str] = mapped_column(String(50), nullable=False)
    tracked_index: Mapped[str] = mapped_column(String(200), nullable=False)
    ter: Mapped[Decimal] = mapped_column(Numeric(6, 4), nullable=False)
    domicile: Mapped[str] = mapped_column(String(50), nullable=False)
    currency_hedged: Mapped[bool] = mapped_column(Boolean, default=False, nullable=False)
    fiscal_year_end: Mapped[str] = mapped_column(String(10), nullable=False)
    german_tax_classification: Mapped[str] = mapped_column(String(50), nullable=False)
    replication_strategy: Mapped[str] = mapped_column(String(50), nullable=False)
    fund_size: Mapped[Decimal | None] = mapped_column(Numeric(18, 2), nullable=True)
    monthly_volume: Mapped[Decimal | None] = mapped_column(Numeric(18, 2), nullable=True)
    volatility_1y: Mapped[Decimal | None] = mapped_column(Numeric(6, 4), nullable=True)
    volatility_3y: Mapped[Decimal | None] = mapped_column(Numeric(6, 4), nullable=True)
    dividend_policy: Mapped[str] = mapped_column(String(50), nullable=False)
    dividend_frequency: Mapped[str | None] = mapped_column(String(20), nullable=True)
    holdings_overview: Mapped[str | None] = mapped_column(Text, nullable=True)
    geographical_distribution: Mapped[dict] = mapped_column(JSONB, nullable=False)
    sector_distribution: Mapped[dict] = mapped_column(JSONB, nullable=False)
    bond_maturities: Mapped[dict | None] = mapped_column(JSONB, nullable=True)
    bond_credit_scores: Mapped[dict | None] = mapped_column(JSONB, nullable=True)
    created_at: Mapped[datetime] = mapped_column(
        DateTime(timezone=True), server_default=func.now(), nullable=False
    )

    holdings: Mapped[list["EtfHolding"]] = relationship(
        back_populates="etf", cascade="all, delete-orphan"
    )
    price_history: Mapped[list["EtfPriceHistory"]] = relationship(
        back_populates="etf", cascade="all, delete-orphan"
    )

    __table_args__ = (
        Index("ix_etfs_geographical_distribution", "geographical_distribution", postgresql_using="gin"),
        Index("ix_etfs_sector_distribution", "sector_distribution", postgresql_using="gin"),
    )


class EtfHolding(Base):
    __tablename__ = "etf_holdings"

    id: Mapped[uuid.UUID] = mapped_column(primary_key=True, default=uuid.uuid4)
    etf_id: Mapped[uuid.UUID] = mapped_column(
        ForeignKey("etfs.id", ondelete="CASCADE"), nullable=False
    )
    company_name: Mapped[str] = mapped_column(String(200), nullable=False)
    weight_pct: Mapped[Decimal] = mapped_column(Numeric(6, 4), nullable=False)
    sector: Mapped[str] = mapped_column(String(100), nullable=False)
    region: Mapped[str] = mapped_column(String(100), nullable=False)
    market_value: Mapped[Decimal | None] = mapped_column(Numeric(18, 2), nullable=True)
    shares: Mapped[Decimal | None] = mapped_column(Numeric(18, 6), nullable=True)

    etf: Mapped["Etf"] = relationship(back_populates="holdings")


class EtfPriceHistory(Base):
    __tablename__ = "etf_price_history"

    id: Mapped[uuid.UUID] = mapped_column(primary_key=True, default=uuid.uuid4)
    etf_id: Mapped[uuid.UUID] = mapped_column(
        ForeignKey("etfs.id", ondelete="CASCADE"), nullable=False
    )
    price: Mapped[Decimal] = mapped_column(Numeric(18, 6), nullable=False)
    currency: Mapped[str] = mapped_column(String(3), nullable=False)
    timestamp: Mapped[datetime] = mapped_column(DateTime(timezone=True), nullable=False)

    etf: Mapped["Etf"] = relationship(back_populates="price_history")

    __table_args__ = (
        Index("ix_etf_price_history_etf_id_timestamp", "etf_id", "timestamp"),
    )
```

### Testing Strategy

No unit tests for this task — migrations and ORM model declarations have no branching logic to unit test.

**Integration test (manual):**

```bash
# From backend/
export DATABASE_URL=postgresql+psycopg://user:pass@localhost:5432/aerarium_test

# Apply migration
alembic upgrade head

# Verify all four tables exist
psql $DATABASE_URL -c "\dt"
# Expected: transactions, etfs, etf_holdings, etf_price_history

# Verify etfs schema (UUID PK, UNIQUE ticker/isin, JSONB columns, GIN indexes)
psql $DATABASE_URL -c "\d+ etfs"

# Verify composite index on etf_price_history
psql $DATABASE_URL -c "\d+ etf_price_history"

# Verify cascade delete: insert etf + holding → delete etf → holding gone
psql $DATABASE_URL -c "
  INSERT INTO etfs (id, ticker, isin, name, issuer, asset_class, tracked_index, ter,
    domicile, currency_hedged, fiscal_year_end, german_tax_classification,
    replication_strategy, dividend_policy, geographical_distribution, sector_distribution)
  VALUES (gen_random_uuid(), 'VWCE', 'IE00B3RBWM25', 'Vanguard FTSE All-World', 'Vanguard',
    'Equities', 'FTSE All-World', 0.0022, 'IE', false, '31/12', 'Accumulating',
    'Full replication', 'Accumulating', '{}', '{}')
  RETURNING id;
"
# Then insert a holding with the returned id, delete the etf, confirm holding count = 0.

# Verify idempotency: running upgrade head again is a no-op
alembic upgrade head

# Verify downgrade
alembic downgrade base
psql $DATABASE_URL -c "\dt"
# Expected: only transactions remains (managed by create_all, not Alembic)
```

**Edge cases:**

- `alembic upgrade head` on a database where tables already exist from a previous run → Alembic checks `alembic_version` table and skips already-applied revisions
- `downgrade base` with FK-constrained child rows present → `ON DELETE CASCADE` handles child deletion automatically before the parent table is dropped
- Missing `DATABASE_URL` env var → `KeyError` raised in `env.py` at import time, before any DDL runs

### Open Questions / Risks

- [ ] **Composite index `DESC` direction**: SQLAlchemy's `Index` does not natively carry `DESC` per-column in `__table_args__`. The composite index on `(etf_id, timestamp DESC)` may need to use `text("timestamp DESC")` inside `op.create_index` in the migration, or `postgresql_ops={"timestamp": "DESC NULLS LAST"}`. Verify correct syntax against PostgreSQL 17 during integration test. **Target:** before merging #39
- [ ] **GIN index in `__table_args__` vs. migration DDL**: The `Index(..., postgresql_using="gin")` in `Etf.__table_args__` informs SQLAlchemy metadata introspection. The authoritative DDL is the migration's `op.create_index(..., postgresql_using="gin")`. Both must be consistent; verify that `alembic check` reports no schema drift after the migration is applied. **Target:** before merging #39
- [ ] **`transactions` table coexistence**: After `alembic upgrade head`, the `transactions` table (created by `create_all()` in `main.py lifespan`) and the three ETF tables (created by Alembic) both exist. Running `alembic downgrade base` removes only the ETF tables. This asymmetry is intentional and documented; a follow-up milestone will establish a baseline migration for `transactions`. Confirm no unintended interaction during integration test. **Target:** before merging #39
